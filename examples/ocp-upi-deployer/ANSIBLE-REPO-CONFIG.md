# Ansible Repository Configuration

## Overview

The OpenShift UPI deployer now supports configurable Ansible playbook repositories, allowing users to specify their own forks or customized versions of the `ocp4-ai-powervm` playbook.

## Changes Made

### 1. Configuration Structure (`types.go`)

Added new `AnsibleConfig` struct to the main `Config` structure:

```go
// AnsibleConfig represents ansible playbook configuration
type AnsibleConfig struct {
	PlaybookRepo   string `yaml:"playbook_repo"`
	PlaybookBranch string `yaml:"playbook_branch"`
	VarsFile       string `yaml:"vars_file"`
}
```

### 2. Main Orchestrator (`main.go`)

Updated the `executeAnsiblePlaybook()` function to use configurable repository settings with sensible defaults:

```go
// Step 1: Clone Ansible playbook repository
playbookRepo := o.config.Ansible.PlaybookRepo
if playbookRepo == "" {
	playbookRepo = "https://github.com/cs-zhang/ocp4-ai-powervm.git"
}
playbookBranch := o.config.Ansible.PlaybookBranch
if playbookBranch == "" {
	playbookBranch = "main"
}

log.Printf("[SSH] Cloning Ansible playbook repository: %s (branch: %s)...", playbookRepo, playbookBranch)
cloneCmd := fmt.Sprintf("cd /root && rm -rf ocp4-ai-powervm && git clone -b %s %s ocp4-ai-powervm", playbookBranch, playbookRepo)
```

### 3. Configuration Files

Added `ansible` section to both configuration files:

#### config-sno.yaml
```yaml
# =============================================================================
# ANSIBLE PLAYBOOK CONFIGURATION
# =============================================================================
# Configuration for the Ansible playbook used to setup the helper node
# and perform OpenShift installation tasks
ansible:
  # Git repository containing the Ansible playbook
  # Default: https://github.com/cs-zhang/ocp4-ai-powervm.git
  # You can specify your own fork or customized version here
  playbook_repo: "https://github.com/cs-zhang/ocp4-ai-powervm.git"
  
  # Git branch to checkout
  # Default: main
  playbook_branch: "main"
  
  # Path to the Ansible variables file (relative to playbook directory)
  # This file will be generated by the deployer
  # Default: vars.yaml
  vars_file: "vars.yaml"
```

#### config.yaml
Same structure as config-sno.yaml

## Usage

### Using Default Repository

If you don't specify the `ansible` section or leave the fields empty, the deployer will use the default upstream repository:

```yaml
# Option 1: Omit the section entirely (uses defaults)

# Option 2: Specify empty values (uses defaults)
ansible:
  playbook_repo: ""
  playbook_branch: ""
  vars_file: ""
```

### Using Custom Fork

To use your own fork or customized version:

```yaml
ansible:
  playbook_repo: "https://github.com/your-username/ocp4-ai-powervm.git"
  playbook_branch: "custom-branch"
  vars_file: "vars.yaml"
```

### Using Different Branch

To use a different branch from the upstream repository:

```yaml
ansible:
  playbook_repo: "https://github.com/cs-zhang/ocp4-ai-powervm.git"
  playbook_branch: "development"
  vars_file: "vars.yaml"
```

## Benefits

1. **Flexibility**: Users can test changes in their own forks before contributing upstream
2. **Customization**: Organizations can maintain internal customizations
3. **Version Control**: Pin to specific branches or tags for reproducible deployments
4. **Development**: Developers can test playbook changes without modifying the deployer code
5. **Backward Compatibility**: Defaults to upstream repository if not specified

## Default Values

| Field | Default Value | Description |
|-------|---------------|-------------|
| `playbook_repo` | `https://github.com/cs-zhang/ocp4-ai-powervm.git` | Upstream repository |
| `playbook_branch` | `main` | Main branch |
| `vars_file` | `vars.yaml` | Ansible variables file name |

## Implementation Details

The deployer:
1. Reads the `ansible` configuration from the YAML file
2. Applies default values if fields are empty
3. Clones the specified repository and branch on the helper node
4. Generates the Ansible variables file
5. Executes the playbook

The clone command removes any existing `ocp4-ai-powervm` directory before cloning to ensure a clean state.

## Testing

To test with a custom repository:

1. Fork the `ocp4-ai-powervm` repository
2. Make your changes in a branch
3. Update `config-sno.yaml` or `config.yaml` with your fork details
4. Run the deployer

The deployer will clone your fork and use it for the deployment.

## Notes

- The repository must be publicly accessible or the helper node must have appropriate SSH keys configured
- The playbook structure must be compatible with the deployer's expectations
- The `vars_file` path is relative to the playbook repository root
- Changes to the playbook may require corresponding changes to the deployer's variable generation logic
